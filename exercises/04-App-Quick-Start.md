# Exercise 4 — Quick Wins Across Your M365 Apps (Session 2A)

- **Apps:** Outlook, Word, Excel, and Teams chat
- **Workflow:** One real Copilot win in each core app, using files already in your OneDrive
- **Material:** **Status_Update.docx**, **Budget_Tracker.xlsx**, **Policy_SOP.pdf** (from pre-work) + your practice threads
- **Estimated time:** 15 minutes
- **Difficulty:** Foundational — the app-by-app tour

---

## Objective

See Copilot do one useful thing in each app you already use, and learn one prompt move that carries everywhere: be specific, frame the audience, set constraints, and ask for the question behind the data.

## Before you start

- [ ] The sample files are in your OneDrive and the practice threads are in your inbox (pre-work).
- [ ] You can open Copilot in Outlook, Word, and Excel. To reference a file or thread, type `/` in the Copilot prompt, start typing its name, and select it.

## What Copilot is doing here

The rhythm is the same in every app: the facilitator demos, you replicate on the referenced file, then refine with one follow-up. Copilot drafts and analyzes; you verify and decide.

---

## Outlook — *audience framing*
In Copilot, type `/`, start typing **DMID Data Call — Response Status**, and select it. Then run:
```text
Summarize this thread for me in 3 bullets: the decision, the deadline, and what I owe next.
```
**Refine:** *"Now rewrite it for my manager, who only needs the risk and the deadline."* Notice how naming the audience changes the result.

## Word — *constraints sharpen output*
In Copilot, type `/`, start typing **Status_Update**, and select it. Then run:
```text
Rewrite this status update in plain language, under 150 words, no acronyms, for a reader outside my team.
```
**Refine:** *"Now make it a 5-bullet version with a one-line bottom line at the top."*

## Excel — *ask for a question, not just an answer*
In Copilot, type `/`, start typing **Budget_Tracker**, and select it. Then run:
```text
Explain what the Contracts tab shows in plain English, and flag anything unusual — a contract near its ceiling, or fully invoiced.
```
**Refine:** *"What would a skeptical reviewer ask about this data?"* Asking for the question often surfaces more than asking for a summary.

## Word / PDF — *ground the answer in the document*
In Copilot, type `/`, start typing **Policy_SOP**, and select it. Then run:
```text
Based on this SOP, what dollar threshold sends a modification to Contracts for approval, and how long does a complex modification take? Quote the part that says so.
```

## Teams — *summarize a chat*
> **GCC note:** live **meeting** recap (Teams Meeting Copilot) isn't available in GCC yet — but Copilot in Teams **chat** is. Open a Teams chat and run:
```text
Summarize this Teams chat: the decision, any action items with owners, and what's still open.
```

---

## Deliverable

One refined Copilot output in each app: an audience-framed email summary, a plain-language status rewrite, a data read-out with outliers flagged, and a grounded answer pulled from the SOP.

## Verify before you rely on it

- [ ] The Excel read-out's figures match the tab — Copilot didn't misread a number.
- [ ] The SOP answer quotes the document and is actually in it (not invented).
- [ ] The rewritten status update kept every fact from the original.

## Key takeaways

- The same `/` reference works in every app — point Copilot at the file or thread, then prompt.
- One prompt move per app: **audience framing**, **constraints**, **ask for the question**, **ground it in the document**.
- Copilot drafts and reads; **you verify the facts** before you use anything.
