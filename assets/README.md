# CCIIO Phase 2 — Sample Assets

Ready-to-use practice files for the Phase 2 workshop (`exercises/`). Hand these out (or drop them in a shared OneDrive/SharePoint folder) so anyone without their own content can still do every lab. **All names, numbers, vendors, and scenarios are fictional** and built around a CMS/CCIIO-style contracting program (COR-led contract modifications, MITG/DMID task orders, Q3 budget review). **Never** substitute real or sensitive CMS data.

## Lab samples (match the exercises)

| File | Use with | What it is |
|---|---|---|
| `SAMPLE-Email-Thread-for-Practice.txt` | Lab 1 — Email → 2-on-2 | Short 3-message Q3 budget thread. Paste into Outlook as a thread, or into Copilot. |
| `SAMPLE-Document-for-PowerPoint.docx` | Lab 2 — Doc → PowerPoint | "Q3 Budget Review & Compliance Update" memo to turn into a 5-slide deck. |
| `SAMPLE-Meeting-Transcript.docx` | Lab 3 — Meeting → SOP | "Contract Modifications Review" transcript (4 speakers) to turn into an SOP. |

## Optional file kit (task-type practice — bring-your-own alternative)

| File | Good for | Why it's built this way |
|---|---|---|
| `Email_Thread.txt` | Summarizing, action-item extraction | 14-message contract-mod thread with decisions and owners **buried in the replies** — practice "catch me up." |
| `Status_Update.docx` | Summarizing, rewriting | Dense, inconsistently formatted bi-weekly status memo — practice tightening and re-formatting. |
| `Budget_Tracker.xlsx` | Data interpretation (Excel) | Three tabs (Contracts, Monthly Spend, Mods Log) of **raw numbers, no narrative** — practice "explain this in plain English / flag outliers." |
| `Quarterly_Deck.pptx` | Editing, condensing | 12 **intentionally overcrowded** slides — practice "make each title a conclusion," trim, and build an executive summary version. |
| `Policy_SOP.pdf` | Q&A, extraction | A process SOP — practice "what does this say about X," extract the decision thresholds, build a quick-reference card. |

## Tips for facilitators
- Pre-load these into a shared OneDrive/SharePoint folder so participants can open them in one click.
- The files are deliberately imperfect (messy formatting, raw data, overcrowded slides, buried decisions) so Copilot has real work to do.
- Remind participants: verify names, dates, dollar figures, and owners against the file before trusting any output.
